# Branding

Source: https://mikrotik.com/logo
Last checked: 2026-04-19. MikroTik page last update: 2026-03-30.

## MikroTik Brand Rules

- Use official MikroTik-provided logo files. Do not redraw, reconstruct, edit, skew, outline, rotate, pattern-fill, recolor, or add shadow/effects to the logo.
- Keep logo as one whole object. Do not use wordmark without symbol.
- Prefer primary logo/mark first. Use alternate versions only when contrast or layout requires it.
- Logo placement works best top-left or bottom-left. Center is acceptable in some contexts.
- Use black logo on light backgrounds and white logo on dark backgrounds. Avoid detailed backgrounds behind logo.
- Clear space should be at least the height of the logo hexagon around all sides.
- Naming: `MikroTik`, `RouterOS`, `WinBox`, `WebFig`, `BackToHome`, `FlashFig`, `BTest`, `Netinstall`, `RouterBOARD`, `The Dude`, `Beacon Manager`, `MikroTik Home`.

## Colors

- MikroTik Black: `#0E0E10`
- MikroTik Gray: `#707372`
- MikroTik Light: `#C8C8C7`
- White: `#FFFFFF`

Complementary gradients from the guide may be used sparingly for product/marketing accents, but application UI should keep the primary brand colors dominant.

## Typography

- Sans-serif: Manrope, weights 300, 400, 700, 800.
- Monospace: JetBrains Mono, weights 300, 400, 700, 800.
- Fallbacks are allowed when local fonts are not installed.

## App Interpretation

- Use a restrained product UI: white/light surfaces, black primary text, gray secondary text, small black action accents.
- Keep components mobile first; expand into grids and split layouts on desktop.
- Reusable layout primitives should support responsive grids instead of one-off page CSS.
